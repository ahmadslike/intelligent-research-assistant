## 🛠️ تحسين Error Messages في الـFrontend

أضفنا 3 رسائل خطأ واضحة بالعربي بدل الرسائل الغامضة:

### قبل التحسين:
```
"Failed to fetch"  ← مش واضح للمستخدم
```

### بعد التحسين:

**1. خطأ الاتصال (Backend مش شغّال):**
```
"تعذّر الاتصال بالخادم. تأكد من تشغيل الـbackend"
```
يظهر لما المتصفح ما يقدر يوصل للـserver أصلاً.

**2. Timeout (60 ثانية بدون رد):**
```
"استغرق الطلب وقتاً طويلاً. حاول مجدداً"
```
استخدمنا `AbortController` — يوقف الطلب تلقائياً بعد 60 ثانية.

**3. خطأ في الـServer (500):**
```
"حدث خطأ في الخادم: " + تفاصيل الخطأ
```

### شو هو AbortController؟

```typescript
const controller = new AbortController();
// بعد 60 ثانية → يوقف الطلب تلقائياً
const timeoutId = setTimeout(() => controller.abort(), 60_000);

const res = await fetch(url, {
  signal: controller.signal  // ربط الـcontroller بالطلب
});

// لو نجح → نوقف المؤقت
clearTimeout(timeoutId);
```

مثل ساعة رمل — لو ما جاء الرد في 60 ثانية، يوقف الانتظار ويطلع رسالة واضحة.

### رسالة الانتظار:

أضفنا نص يظهر أثناء البحث:
```
"جارٍ البحث... قد يستغرق دقيقة أو أكثر"
```

عشان المستخدم يعرف إن البحث طبيعي يأخذ وقت.
