## 🎯 ما الذي تعلمناه اليوم؟

اليوم تعلمنا كيف "نتحكم" في Claude Code نفسه — مش بس نستخدمه، بل نديره بذكاء. مثل ما تعلمت أمس كيف تحكي مع الكمبيوتر، اليوم تعلمت كيف تتحكم بـ"المساعد" نفسه.

---

### 1. شو هي الـSession؟

تخيّل إنك تتصل بصديق بالتلفون. كل مكالمة = session جديدة. لما تقطع وترجع تتصل، صديقك نسي كل اللي قلته في المكالمة السابقة — إلا لو عندك "رقم مرجعي" للمكالمة القديمة.

Claude Code نفس الشي:
- كل مرة تكتب `claude` = جلسة جديدة (Claude يبدأ من صفر)
- بس عند الخروج يعطيك **Session ID** — رقم فريد تقدر فيه تكمل من حيث توقفت

**مثال عملي:**
```
Resume this session with:
claude --resume 78156854-314d-409e-a306-67b8ffd8038d
```

---

### 2. شو هي الـPermissions (الصلاحيات)؟

تخيّل إنك وظّفت مساعد في مكتبك. ما بتخليه يمسح ملفاتك بدون ما يسألك أولاً — حتى لو قلتله "نظّف المكتب". هاد هو نظام الـPermissions في Claude Code.

**قبل أي أمر "خطير"، Claude يسألك:**
```
This command requires approval
Do you want to proceed?
1. Yes
2. Yes, and don't ask again for this session
3. No
```

**متى تختار كل خيار؟**
- **1. Yes** → لو الأمر عادي وتثق فيه لكن تبي تتحقق من كل خطوة
- **2. Yes, allow all** → لو شغّال على مهمة واضحة وتبي توفر الوقت
- **3. No** → لو الأمر خطير أو مش متأكد

---

### 3. شو هو Plan Mode؟

تخيّل إنك طلبت من البنّاء يعمّر غرفة جديدة. في طريقتين:
- **بدون Plan Mode:** يبدأ يبني مباشرة بدون ما يسألك
- **مع Plan Mode:** يرسم لك الخطة أولاً وينتظر موافقتك قبل ما يبدأ

Plan Mode يخلي Claude يشرح خطته كاملة قبل ما ينفّذ أي شي.

**طريقتان لتفعيله:**

**الطريقة 1 — مرة وحدة:**
```
Before doing anything, explain your full plan and wait for my approval. Then [طلبك هنا]
```

**الطريقة 2 — دائم في الجلسة (الأذكى):**
اضغط `Shift+Tab` — يظهر في الأسفل:
```
? plan mode on (shift+tab to cycle)
```

---

### 4. Thinking Modes — مستويات التفكير

Claude Code يمكنه يفكر بمستويات مختلفة، زي ما الإنسان يفكر بسرعة لو السؤال بسيط، أو يفكر عميق لو المسألة معقدة.

```
/effort low     → أسئلة بسيطة، يوفر tokens
/effort medium  → الافتراضي، 90% من الوقت
/effort high    → مشاكل معقدة، يستهلك أكثر
```

**قاعدة:** ابدأ دائماً بـmedium. لو ما حل المشكلة، جرّب high.

---

### 5. شو هو `/context`؟

تخيّل إن Claude عنده "ذاكرة عمل" محدودة — مثل سطح مكتبك. لو امتلأ، ما يقدر يحفظ معلومات جديدة. `/context` يعرضلك كم استهلكت من هذه الذاكرة.

**ماذا يعرض؟**
```
16.7k/200k tokens (8%)
Free space: 150.3k (75.2%)
```
= استهلكت 8% فقط، لسا عندك 75% فاضي ✅

---

### 6. شو هو `/compact`؟

لما المحادثة تطول، `/compact` يضغطها لملخص مختصر بدون ما تخسر السياق المهم.

مثال: محادثة من 100 رسالة تصير ملخص من 5 أسطر — بس Claude لسا يتذكر الأهم.

---

### 7. شو هو `.gitignore`؟

تخيّل إن GitHub هو صندوق بريد عام — أي شخص يقدر يفتحه ويشوف محتواه. `.gitignore` هو قائمة الأشياء اللي ما تبيها تنرسل في هاد الصندوق (مثل: كلمات السر، مفاتيح API).

**مثال على محتواه:**
```
# لا ترفع هذه الملفات لـGitHub أبداً
.env
api-keys.txt
*.key
node_modules/
```

### 8. شو هو CLAUDE.md المحدّث؟

أضفنا للملف تفاصيل المشروع الكاملة:

- الـ4 Agents ومهمة كل واحد
- ملاحظة إن المطوّر (أحمد) مبتدئ — حتى Claude يشرح بوضوح في كل جلسة

## 🛠️ كل الأوامر التي استخدمناها اليوم

---

**الأمر:**
```
/cost
```
**شو يعني؟** "كم صرفت من رصيدي؟"  
**وظيفته:** يعرض حالة الاستهلاك — هل تستخدم Pro أو API  
**الناتج:** `You are currently using your subscription to power your Claude Code usage`  
**ملاحظة:** مع Pro = مجاني، مع API = بيحسب بالـ$

---

**الأمر:**
```
/context
```
**شو يعني؟** "كم استهلكت من ذاكرة هاي الجلسة؟"  
**وظيفته:** يعرض استهلاك الـtokens بالتفصيل + الأدوات المتاحة (MCPs)  
**متى تستخدمه؟** لما تحس الجلسة ثقيلة أو بطيئة  
**الناتج المهم:**
```
16.7k/200k tokens (8%)
Free space: 150.3k (75.2%)
```

---

**الأمر:**
```
/compact
```
**شو يعني؟** "لخّص المحادثة واحتفظ بالمهم"  
**وظيفته:** يضغط تاريخ المحادثة ويحرّر مساحة في الـcontext  
**متى تستخدمه؟** كل ساعتين تقريباً في جلسات طويلة  
**الناتج:** `Compacted (ctrl+o to see full summary)`  
**ملاحظة:** لا تفقد السياق المهم — فقط يضغطه

---

**الأمر:**
```
/clear
```
**شو يعني؟** "امسح كل شي وابدأ من صفر"  
**وظيفته:** يمسح كامل تاريخ المحادثة  
**متى تستخدمه؟** لما تنهي موضوع كامل وتبدأ موضوع مختلف تماماً  
**تحذير:** لا يمكن التراجع عنه

---

**الأمر:**
```
/review
```
**شو يعني؟** "راجع الكود الحالي"  
**وظيفته:** يحلل الكود ويقترح تحسينات  
**ملاحظة:** يحتاج `gh` (GitHub CLI) لبعض وظائفه

---

**الأمر:**
```
/effort medium
/effort high
/effort low
```
**شو يعني؟** "حدّد مستوى تفكيرك"  
**وظيفته:** يغير عمق تحليل Claude  
**الافتراضي:** medium  
**متى high؟** Architecture design, debugging معقد, قرارات مهمة

---

**الأمر (اختصار لوحة المفاتيح):**
```
Shift + Tab
```
**شو يعني؟** تبديل Plan Mode on/off  
**وظيفته:** يفعّل/يوقف وضع التخطيط لكل الجلسة  
**يظهر في الأسفل:** `? plan mode on (shift+tab to cycle)`

---

**الأمر (اختصار لوحة المفاتيح):**
```
Ctrl + C
```
**شو يعني؟** خروج سريع  
**وظيفته:** إيقاف العملية الحالية أو الخروج من Claude Code  
**ملاحظة:** أسرع من كتابة `/exit`

---

**الأمر:**
```
claude --resume [SESSION_ID]
```
**شو يعني؟** "كمّل من حيث وقفت"  
**وظيفته:** يستأنف جلسة سابقة مع كامل سياقها  
**متى تستخدمه؟** لو أغلقت PowerShell وتبي تكمل نفس الجلسة

## ❓ الأسئلة التي سألتها خلال اليوم + الأجوبة

**السؤال:** أختار 1 أو 2 لما Claude يطلب إذن؟  
**الجواب:** 2 (Yes, allow all) لو واثق بالمهمة وتبي توفر وقت. 1 (Yes) لو تبي تتحقق من كل خطوة.  
**ليش مهم:** توفير وقت مقابل تحكم أكبر — اختار حسب الموقف.

---

**السؤال:** PowerShell ما يدعم العربية؟  
**الجواب:** صح، PowerShell يعرض العربية كـ"؟؟؟" — دائماً اكتب بالإنجليزي في Terminal.  
**ليش مهم:** قاعدة أساسية — Terminal = إنجليزي دائماً.

## 🧠 أسئلة الاختبار في نهاية اليوم + الأجوبة

**السؤال 1:** شو الفرق بين `/compact` و `/clear`؟  
**جوابي:** `/compact` للضغط مع الحفاظ على السياق كل ساعتين. `/clear` لمسح كل شي عند انتهاء الجلسة  
**الجواب الصحيح:** ✅ صح تماماً

---

**السؤال 2:** شو يعني Plan Mode وكيف تفعّله بسرعة؟  
**جوابي:** يعني Claude يشرح خطته وينتظر موافقتي. أفعّله بـShift+Tab  
**الجواب الصحيح:** ✅ صح تماماً — واكتشفت Shift+Tab بنفسك!

---

**السؤال 3:** لو Claude طلب إذن لأمر خطير، شو تختار؟  
**جوابي:** No  
**الجواب الكامل:** No للأوامر الخطيرة ✅ بس لو الأمر عادي وتثق فيه → Yes أو Yes allow all

---

**السؤال 4:** شو هو Session ID وليش مفيد؟  
**جوابي:** رقم فريد للجلسة أقدر أرجع فيه لنفس الجلسة  
**الجواب الصحيح:** ✅ ممتاز

---

**السؤال 5:** متى تستخدم `/effort high` ومتى `medium`؟  
**جوابي:** medium للأغلب، high للمهام المعقدة  
**الجواب الصحيح:** ✅ صح

## 📊 ملخص سريع (للمراجعة قبل يوم 3)

### الصفحة 1 من 2: الأوامر والمفاهيم السريعة

#### أهم الأوامر (cheat sheet):

| الأمر | شو يعمل |
|-------|---------|
| `/cost` | شوف الاستهلاك |
| `/context` | شوف حالة الذاكرة |
| `/compact` | ضغط المحادثة |
| `/clear` | مسح كامل |
| `/effort medium/high/low` | مستوى التفكير |
| `Shift+Tab` | تفعيل Plan Mode |
| `Ctrl+C` | خروج سريع |
| `claude --resume ID` | كمّل جلسة سابقة |

#### أهم 5 مفاهيم:

1. **Session:** كل تشغيل لـClaude = جلسة جديدة. Session ID يخليك تكمل من حيث توقفت.

2. **Permissions:** Claude يطلب إذنك قبل الأوامر الخطيرة — Yes/Yes allow all/No.

3. **Plan Mode:** Shift+Tab يخلي Claude يشرح خطته قبل التنفيذ.

4. **Thinking Modes:** medium للعادي، high للمعقد، low للبسيط.

5. **.gitignore:** قائمة الملفات اللي ما ترفعها لـGitHub أبداً (API keys, .env).
