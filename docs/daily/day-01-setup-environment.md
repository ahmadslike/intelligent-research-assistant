## 🎯 ما الذي تعلمناه اليوم؟

تخيّل إنك تريد تتكلم مع الكمبيوتر — مش بالنقر على الأيقونات، لا، بالكتابة المباشرة. زي ما تكتب رسالة لصديقك بالضبط، بس صديقك هنا هو الكمبيوتر. هاد اليوم تعلمنا كيف "نحكي" مع الكمبيوتر، وكيف نحفظ شغلنا، وكيف نستخدم أذكى مساعد برمجة موجود.

---

### 1. شو هو Terminal؟

تخيّل إنك في مطبخ. عندك طريقتين تطلب فيهم من الطاهي:
- **الطريقة الأولى (GUI):** تفتح قائمة الطعام، تضغط على صورة الطبق، وتنتظر
- **الطريقة الثانية (Terminal):** تقول مباشرة "جيبلي طبق رقم 5 بدون ملح"

Terminal هو الطريقة الثانية — تكتب أوامر مباشرة للكمبيوتر بدل ما تضغط على أيقونات. أسرع وأقوى بكثير.

---

### 2. شو هو CLI؟

CLI = Command Line Interface = واجهة سطر الأوامر.

هو اسم ثاني لـTerminal. لما تسمع "CLI" يعني "المكان اللي بتكتب فيه أوامر نصية للكمبيوتر". مثلاً: PowerShell هو CLI على Windows.

الفرق بين GUI و CLI:
- **GUI:** تضغط على أيقونة لتفتح برنامج
- **CLI:** تكتب اسم البرنامج وتضغط Enter

---

### 3. شو هو Git وشو الفرق بينه وبين GitHub؟

**Git** مثل آلة الزمن لكودك. تخيّل إنك كتبت تقرير مدرسي وبدك تحفظ نسخ منه:
- نسخة يوم الإثنين
- نسخة بعد ما عدّلت الفقرة الأولى
- نسخة نهائية

Git بيعمل هيك بالضبط، بس للكود. كل مرة تعمل "commit" = تحفظ نسخة جديدة.

**GitHub** مثل Google Drive بس للكود. Git يحفظ النسخ على جهازك، GitHub يرفعها على الإنترنت عشان:
- تحمي شغلك (لو كسر جهازك، كودك محفوظ)
- تشاركه مع الناس
- يظهر في Portfolio تاعتك

**الخلاصة:** Git = الأداة على جهازك، GitHub = الموقع على الإنترنت

---

### 4. شو هو Node.js؟

تخيّل إن JavaScript لغة تشتغل بس داخل المتصفح (Chrome, Firefox). Node.js مثل "مولّد كهرباء" يخلي JavaScript تشتغل خارج المتصفح، على جهازك مباشرة.

نحن محتاجينه عشان Claude Code CLI مكتوب بـJavaScript وما يشتغل بدونه.

---

### 5. شو هو Claude Code CLI؟

Claude Code CLI هو نسخة Claude الذكية اللي تشتغل **داخل مجلد مشروعك**. الفرق بينه وبين Claude العادي (اللي إحنا نحكي فيه هلق):

- **Claude العادي (هاد الشات):** تسأله أسئلة عامة، يجاوبك
- **Claude Code CLI:** يفتح داخل مجلدك، يقرأ ملفاتك، يكتب كود مباشرة فيهم، يشغّل أوامر

مثل الفرق بين تسأل مهندس بالتلفون أو تجيبه عالبيت يشتغل معك مباشرة.

---

### 6. شو هو CLAUDE.md؟

هو أول ملف يقرأه Claude Code لما تشغّله في مشروعك. زي "بطاقة تعريف" للمشروع. فيه:
- شو هو المشروع
- كيف يساعدك
- القواعد اللي تتبعها

بدونه Claude Code ما بيعرف شي عن مشروعك.

---

### 7. شو هو commit؟

تخيّل إنك تلعب فيديو جيم وضغطت Save. الـcommit هو نفس الفكرة — حفظ نقطة معينة في تاريخ مشروعك مع رسالة تقول "شو عملت هنا".

مثال: `git commit -m "Add login page"` = "احفظ نسخة وقول إني أضفت صفحة تسجيل دخول"

---

### 8. شو هو push؟

بعد ما تعمل commit (حفظ على جهازك)، الـpush يرفع هاد الحفظ لـGitHub. مثل ما تحفظ ملف على جهازك ثم ترفعه لـGoogle Drive.

---

### 9. شو هو Personal Access Token؟

GitHub ما بيقبل كلمة سرك لما ترفع كود من Terminal لأسباب أمنية. بدل كذلك، تعمل "token" — كلمة سر خاصة لجهازك بس. لو سُرق الـtoken، تلغيه بدون ما تغير كلمة سرك الأساسية.

---

### 10. شو هو Execution Policy؟

Windows بشكل افتراضي يمنع تشغيل scripts (برامج نصية) خوفاً من الفيروسات. `Set-ExecutionPolicy` هو الأمر اللي يقول لـWindows "ثق بـscripts موقّعة من مصادر موثوقة".

## ❓ الأسئلة التي سألتها خلال اليوم + الأجوبة

**السؤال:** أي نافذة أستخدم — CMD أو PowerShell؟  
**الجواب:** PowerShell دائماً. CMD قديم ومش كافي للتطوير الحديث. PowerShell أقرب لـLinux وبيدعم كل الأوامر اللي نحتاجها.  
**ليش مهم:** لأن معظم الأدوات الحديثة تشتغل أفضل مع PowerShell

---

**السؤال:** ليش npm ما اشتغل بس node اشتغل؟  
**الجواب:** Windows عنده سياسة أمان تمنع تشغيل scripts تلقائياً. node برنامج عادي، بس npm هو script. الحل: تغيير ExecutionPolicy.  
**ليش مهم:** هاي مشكلة شائعة جداً على Windows، لازم تعرف حلها

---

**السؤال:** لما أختار Yes أو Yes allow all edits في Claude Code؟  
**الجواب:** "Yes allow all edits during this session" أريح — ما تضطر توافق على كل تعديل. بس لو خايف على ملفات مهمة، اختار "Yes" العادية.  
**ليش مهم:** توفير وقت وتجنب الإزعاج المتكرر

## 🐛 الأخطاء التي واجهتنا وكيف حللناها

**الخطأ:**
```
npm : File C:\Program Files\nodejs\npm.ps1 cannot be loaded because 
running scripts is disabled on this system.
CategoryInfo: SecurityError
FullyQualifiedErrorId: UnauthorizedAccess
```
**ليش ظهر؟** Windows يمنع تشغيل PowerShell scripts بشكل افتراضي كإجراء أمني ضد الفيروسات.  
**كيف حللناه؟**
1. فتحنا PowerShell كـAdministrator
2. كتبنا: `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`
3. وافقنا بكتابة `Y`

**الدرس المستفاد:** لو شفت خطأ "running scripts is disabled" على Windows، هاد هو الحل دائماً.

---

**الخطأ:** أُغلقت نافذة PowerShell بالغلط في منتصف الشغل  
**ليش ظهر؟** ضغط غلط على زر الإغلاق  
**كيف حللناه؟** فتحنا PowerShell من جديد وكتبنا `cd C:\Users\HP\Desktop\intelligent-research-assistant` للرجوع للمشروع  
**الدرس المستفاد:** PowerShell مجرد نافذة — إغلاقها ما يضرر شي. الكود محفوظ في الملفات دائماً.

---

**الخطأ:** في ملف `api-keys.txt` كان مكتوب `GITHUB_API_KEYghp_...` بدون `=`  
**ليش ظهر؟** خطأ في الكتابة اليدوية  
**كيف حللناه؟** فتحنا الملف وأضفنا `=` ليصير `GITHUB_API_KEY=ghp_...`  
**الدرس المستفاد:** ملفات الإعدادات حساسة جداً للصيغة — دائماً `KEY=VALUE`

## 🧠 أسئلة الاختبار في نهاية اليوم + الأجوبة

**السؤال 1:** شو الفرق بين `git add` و `git commit`؟  
**جوابي الأولي:** git add للإضافة و git commit لحفظ التعديلات والملفات  
**الجواب الصحيح الكامل:** `git add` يضع الملفات في "منطقة انتظار" (staging area) — زي ما تحط أشياء في صندوق قبل ما تبعثه. `git commit` يحفظ كل اللي في الصندوق كنسخة دائمة مع رسالة تصف شو عملت.  
**ملاحظة:** الجواب صح في جوهره ✅

---

**السؤال 2:** شو يعني `cd ..`؟  
**جوابي الأولي:** يعني بقول للبوشيل افتح ملف كذا  
**الجواب الصحيح الكامل:** `cd ..` يعني "ارجع للمجلد اللي فوقي". النقطتين `..` تعنيان "المجلد الأب". مثلاً لو في `Desktop\project` وكتبت `cd ..` رح تروح لـ`Desktop`.  
**ملاحظة:** الجواب خطأ ✗ — تم التصحيح فوراً

---

**السؤال 3:** ليش استخدمنا PowerShell بدل CMD؟  
**جوابي الأولي:** لأن CMD ما فيه كل الأشياء اللي نحتاجها والأوامر مش حتتفعل  
**الجواب الصحيح الكامل:** PowerShell أحدث وأقوى من CMD، يدعم أوامر شبيهة بـLinux، ومتوافق مع Node.js وGit وClaude Code. CMD قديم من التسعينات ومحدود الإمكانيات.  
**ملاحظة:** الجواب صح ✅

---

## 📊 ملخص سريع (للمراجعة قبل يوم 2)

### الصفحة 1 من 2: الأوامر والمفاهيم السريعة

#### أهم 12 أمر تعلمتها اليوم (cheat sheet):

| الأمر | شو يعمل |
|-------|---------|
| `pwd` | وين أنا؟ يطبع المسار الحالي |
| `ls` | شو في المجلد؟ يعرض المحتويات |
| `cd اسم-المجلد` | ادخل لمجلد |
| `cd ..` | ارجع للمجلد الأب |
| `mkdir اسم` | أنشئ مجلد جديد |
| `git init` | ابدأ تتبع Git في المجلد |
| `git add .` | جهّز كل التغييرات للحفظ |
| `git commit -m "رسالة"` | احفظ نسخة مع رسالة |
| `git push` | ارفع لـGitHub |
| `git status` | شو الوضع؟ شو تغيّر؟ |
| `claude` | شغّل Claude Code |
| `/init` | أنشئ/حدّث CLAUDE.md |

#### أهم 5 مفاهيم (في جملتين لكل مفهوم):

1. **Terminal/CLI:** مكان تكتب فيه أوامر نصية للكمبيوتر بدل الضغط على الأيقونات. PowerShell هو الـCLI اللي نستخدمه على Windows.

2. **Git:** أداة لحفظ نسخ من كودك على جهازك. كل مرة تعمل commit = تحفظ snapshot يمكنك الرجوع له.

3. **GitHub:** موقع يخزّن كودك على الإنترنت. Git للمحلي، GitHub للسحابة.

4. **Claude Code CLI:** نسخة Claude تشتغل داخل مجلد مشروعك، تقرأ ملفاتك وتكتب كود مباشرة فيهم.

5. **CLAUDE.md:** ملف التعليمات لـClaude Code — يقرأه أول ما تشغّله ليفهم شو المشروع.
